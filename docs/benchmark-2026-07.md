## Task 04 で Mesh が踏んだ2つの往復(最重要の収穫)

**往復1: 複数行unionの行頭`|`継続が書けない(負転移・言語側の穴)**
被験体はTSの定番フォーマットで判別可能unionを複数行に折った:

```
type Expr = { kind: "num", value: int }
    | { kind: "add", left: Expr, right: Expr }
    | ...
```

Meshは改行を文末とみなすため1行目で宣言が完結し、`use 'struct Expr { ... }' to define a
data shape` エラーになった。**union自体の理解は完璧で、フォーマットだけで落ちた**。
TSは行頭`|`継続を許すので、これはF-6系(TS学習データとの負転移)のフォーマット版。
→ **✅ 2026-07-20対応済み**: パーサが型宣言中の「改行(ASIの`;`)の先の`|`」を継続として
読むようになり、行頭`|`・行末`|`の両フォーマットが動く。カードの判別可能union例も
複数行形に変更して実例化した

**往復2: 関数名 `eval` が予約語**
仕様が関数名を`eval`と指定していたため、`RESERVED`(JS strict mode由来、カード実験第6回で
追加)に当たった。エラーメッセージ(`'eval' is a reserved word and cannot be used as a name`)
から1往復で`evaluate`へ全箇所リネームして解決 — 「修正方法が分かるエラー」は機能した。
→ **✅ 2026-07-20対応済み**: RESERVED全34語をカードのBindings節に明記した。
⚠️ 計測の正直な注記: TS被験体のプロンプトにだけ「evalが予約されていれば別名で可」の
注記があった(TSもstrict modeで`eval`を関数名にできないため防衛的に入れたもの)。
Mesh/Goには無し。Goは`eval`が有効なので影響なし、Meshはこの注記があれば往復2は
起きなかった可能性がある — 次回はプロンプトを完全対称にすること

## 質的所見(Meshの生成コードのイディオム充実度)

白紙AI+カードだけで、直近実装した機能群が**自然に・正しく**使われた:

- **Task 01**: `Todo[]{}`空配列・`push`・structが参照値であること(ループ内`t.done = true`が
  呼び出し元に見える)を正しく利用
- **Task 02**: `toInt` → `is error`早期return → 残りがintに絞られて範囲比較(narrowing連鎖)。
  `!(cfg.port is none)`(F-6のド・モルガン形)+フィールドパス条件を自然に書いた —
  **TSの感覚で書く負転移パターンが実際に出現しており、F-6が地雷を除去した形の実証**
- **Task 03**: `spawn`×5 → `<-task`×5 → 順序回復・集約。受信型が`T | closed`になることまで
  理解し、ヘルパー関数の引数型を`int | error | closed`と明示
- **Task 04**(最終形): 自己参照する判別可能union・部分構造パターンmatch・アーム内narrowing・
  再帰2関数。3言語中最短の34行(TS 50・Go 64)

## 仮説に対する現時点の答え

- **「学習データゼロの言語+カードが、学習データ豊富なTS/Goと互角」は概ね成立**。
  12セル中11セルで完全互角、差がついた1セルもフォーマット規約と予約語という
  「浅い」層の問題で、修正はエラーメッセージ経由で即完了した
- 天井効果に注意: Sonnetは3言語とも強く、この4タスクでは差がほぼ出ない。
  モデルを弱くする(Haiku)・タスクを大きくする・試行数を増やすのが次の感度向上策
- 負転移は「機能の使い方」の層ではもう観測されず(F-1〜F-6対応後)、
  **フォーマット・予約語の層に残っている** — 修正コストが安い層まで掃除が進んだとも言える

## ハーネス運用メモ(再計測する人へ)

- TS検査はリポジトリ内から実行するなら`--ignoreConfig`必須(tsconfig.jsonと競合、TS 7仕様)
- macOSでビルド直後のGoバイナリ初回実行は署名検証で+300ms以上遅れる。
  壁時間を測る前に1回空実行(pre-warm)するか2回目以降を採用する
- Mesh/Goは事前ビルドした成果物で壁時間を測る(コンパイル時間の混入を避ける)
- 被験体プロンプトは3言語で完全対称にする(今回の`eval`注記の非対称を繰り返さない)
